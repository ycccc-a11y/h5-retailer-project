# 手机无法访问 - 故障排查指南

## ✅ 已确认正常
- ✅ 前端服务器已启动（端口8080）
- ✅ 后端API已启动（端口5001）
- ✅ IP地址：192.168.1.35

## 🔍 排查步骤

### 1. 检查手机和电脑是否在同一WiFi
- 打开手机WiFi设置
- 确认连接的是同一个WiFi网络
- **重要**：手机不能用4G/5G，必须用WiFi

### 2. 在手机上测试访问
打开手机浏览器，尝试访问：
```
http://192.168.1.35:8080
```

**常见错误提示：**
- "无法连接到服务器" → 防火墙或网络问题
- "连接超时" → 防火墙阻止了端口
- "无法访问此网站" → 检查WiFi网络

### 3. 检查Windows防火墙（最常见问题）

#### 方法1：运行防火墙配置脚本（推荐）
双击运行 `配置防火墙.bat`

#### 方法2：手动配置防火墙
1. 打开"Windows Defender 防火墙"
2. 点击"高级设置"
3. 点击"入站规则" → "新建规则"
4. 选择"端口" → 下一步
5. 选择"TCP"，特定本地端口：`8080,5001`
6. 选择"允许连接"
7. 应用到所有配置文件
8. 完成

### 4. 测试连接
在手机浏览器中访问：
```
http://192.168.1.35:8080
```

如果还是不行，尝试：
```
http://192.168.1.35:8080/index.html
```

### 5. 检查后端API是否可以从手机访问
在手机浏览器访问：
```
http://192.168.1.35:5001/api/search/license?page=1&page_size=1
```
如果能看到JSON数据，说明后端正常。

### 6. 临时关闭防火墙测试（仅用于测试）
1. 打开"Windows Defender 防火墙"
2. 点击"启用或关闭Windows Defender防火墙"
3. 临时关闭防火墙（仅测试用）
4. 测试手机访问
5. **测试完后记得重新开启防火墙！**

## 🔧 其他可能的问题

### 问题1：路由器阻止了设备间通信
- 某些路由器有"AP隔离"功能，会阻止设备间通信
- 检查路由器设置，关闭"AP隔离"或"客户端隔离"

### 问题2：IP地址变化
- 如果重新连接WiFi，IP地址可能变化
- 重新运行 `ipconfig | findstr IPv4` 获取新IP

### 问题3：浏览器缓存
- 清除手机浏览器缓存
- 尝试使用无痕/隐私模式

### 问题4：杀毒软件阻止
- 某些杀毒软件会阻止端口访问
- 临时关闭杀毒软件测试

## 📱 推荐测试步骤
1. 先在同一WiFi下用电脑浏览器访问 `http://192.168.1.35:8080` 测试
2. 如果电脑能访问，再用手机访问
3. 如果电脑也访问不了，检查防火墙
